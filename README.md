# База данных для управления информационными потоками в судебной системе

## Описание проекта

Данная база данных предназначена для управления информационными потоками в судебной системе. Она обеспечивает хранение и обработку информации о юридических и физических лицах, судебных делах, документах, судьях и их взаимосвязях.

## Структура файлов

- `01_create_database.sql` - SQL скрипт для создания структуры базы данных (DDL)
- `02_insert_initial_data.sql` - SQL скрипт для вставки тестовых данных
- `03_queries.sql` - SQL запросы для выполнения аналитических задач (пункт 5 задания)
- `04_dml_operations.sql` - SQL операции для добавления, обновления и удаления данных (пункты 6-7 задания)
- `DATABASE_DOCUMENTATION.md` - Подробная документация по структуре базы данных и таблицам (пункт 8 задания)
- `USAGE_GUIDE.md` - Руководство по использованию базы данных
- `README.md` - Общее описание проекта

## Установка и настройка

1. Убедитесь, что у вас установлен MySQL или MariaDB
2. Выполните скрипт создания базы данных:
   ```bash
   mysql -u root -p < 01_create_database.sql
   ```
3. Загрузите тестовые данные:
   ```bash
   mysql -u root -p < 02_insert_initial_data.sql
   ```

## Структура базы данных

### Диаграмма связей

База данных состоит из следующих основных таблиц:

- **judicial_districts** - Судебные округа
- **case_categories** - Категории дел
- **individuals** - Физические лица
- **legal_entities** - Юридические лица
- **judges** - Судьи
- **court_cases** - Судебные дела
- **participant_roles** - Роли участников процесса
- **case_participants_individuals** - Связь дел с физическими лицами
- **case_participants_legal_entities** - Связь дел с юридическими лицами
- **court_documents** - Судебные документы
- **case_decisions** - Решения по делам
